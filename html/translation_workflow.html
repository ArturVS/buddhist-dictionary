Translation and Document Analysis Workflows

This page describes the ways that the web site can be used in different translation workflows. Besides translations, document analysis also includes

  1. Reading a document, historic or modern, and using the web site to understand more about certain words or phrases.
  2. Getting a sense of an untranslated historic document. Full translation may not be feasible for but you want to get an overall sense of the content and nature of the document.
  3. Commentary or annotations in a modern language, say for research of a historic Chinese document. The output of the research may be a paper in modern Chinese or English. Additional notes may be create about the meanings of the words, the identity of certain people or places, or notes about difficult phrases.
  4. Translation memory. That is, once a new translation term or phrase has been discovered the user wants to record it so that it can be easily identified next time it is encountered.
  5. Reuse material found on the web site.

Casual use for a private translation

In this workflow a user does not join the GitHub project but simply wants to use the web site as tool for a private translation. The user has their own private copy of the document and does not wish to share it. The user wants to use the site to search word meanings and for translation memory.

  1. Phrases or words are cut-and-paste from the private Word document into the word or phrase search page to find the word meanings.
  2. If a word is found that is not in the dictionary then the user can open a GitHub issue describing the word to add. The content will periodically be incorporated into the dictionary by a project member and the web site refreshed.
  3. The user can copy material from the web site under the terms of the Creative Commons Attribution-Share Alike 3.0 License license.

Use to get the sense of a historic Chinese document

In this workflow the user does not have a deep knowledge of Chinese but wants to get the overall sense of a document. The user is not a project member.

  1. If the document is short then phrases are cut-and-paste from the private Word document into the word or phrase search page to find the tabulated list of word meanings.
  2. If the document is long the user downloads the project from and uses the command line tools to analyze the text. Please note that the command line tools are not complete yet.

Project team member for a publicly available translation

In this workflow a user joins the GitHub project, uses the web site as tool for translation, and contributes the content back to the project.

  1. A new corpus document will be created. The orginal text will be saved as a plain text document and a mark down document will be created to save links to words and commments about the content of the source document.
  2. Phrases or words are cut-and-paste from the source document into the word search to find the word meanings. As new words and additional information about existings words are discovered the project member adds these directly to the text-based data files.
  3. The GitHub content will periodically be incorporated into the web site by the project member resulting in a the web site refresh.
